# GAME_SPEC.md — ぼうけんドリル ゲーム仕様書

現行 v5 の完全な仕様。実装の「正」はコードだが、意図と全体像はここを参照すること。

---

## 1. コンセプト

- 対象：小学1年生（学習が早いため2〜3年内容も先取り）
- 科目：算数・国語
- 形式：島を探検する冒険の文脈で、ドリルを進めるとステージが解放される
- 学習スタイル：**ドリル × ゲーム世界観のバランス型**
- 音：**控えめ**（効果音のみ、ON/OFF可。BGMなし）
- 対象デバイス：iPad（PWA、全画面、オフライン動作）

---

## 2. 画面構成

```
[ぼうけんマップ]  タブ1 … 島を縦にスクロール。17ステージがジグザグに配置
    └ ステージノードをタップ
        └ [導入モーダル]  ナビキャラの一言（同日1回のみ）
            └ [クイズ画面]  出題 → 即時フィードバック（オーバーレイ）
                └ [結果画面]  星評価・宝物発見演出
[ずかん]        タブ2 … 集めた宝物17種と発見メモ
[おうちのひと]   タブ3 … PIN(1234) → 進捗ダッシュボード・バックアップ・リセット・キャラ改名
```

### 初回起動
ナビキャラ（コンパス）の**命名ダイアログ**が出る。子どもが名前を付ける（最大6文字、
未入力なら「コンパスくん」）。以降、全セリフが `【なまえ】「…」` の形式で表示される。

### ステージ解放
`isUnlocked(i) = (i === 0) || STAGES[i-1] がクリア済み` の順次解放。
クリア条件：**出題数の60%以上正解**。

### 星評価
- ★★★：全問正解
- ★★☆：(n-2)問以上
- ★☆☆：60%以上（クリアライン）

## 4. 出題ロジック一覧（層1）

| 関数 | 内容 | 出題形式 | 値域 |
|---|---|---|---|
| `genCompare()` | 数の大小 | 2択 | 1〜20 |
| `genAdd10()` | たし算（繰上なし） | 数値入力 | 答え 2〜10 |
| `genSub10()` | ひき算（繰下なし） | 数値入力 | 答え 1〜9 |
| `genAddCarry()` | たし算（繰上あり） | 数値入力 | 答え 11〜18 |
| `genSubBorrow()` | ひき算（繰下あり） | 数値入力 | 被減数 11〜18 |
| `genKuku()` | 九九 | 数値入力 | 1×1〜9×9 |
| `genHissanAdd()` | 2桁のたし算 | 数値入力 | 10〜99 同士 |
| `genHissanSub()` | 2桁のひき算 | 数値入力 | 答えが負にならない |
| `genDiv()` | わり算 | 数値入力 | 商・除数とも 2〜9、あまりなし |
| `genBigNum()` | 大きな数 | 2択 or 数値入力 | 100〜999。大小比較 or 位の合成 |
| `genClock()` | 時計よみ | 4択 | 1〜12時、なんじ／なんじはん。**問題文にアナログ時計SVGを埋め込む** |
| `makeKanjiGen(poolName, prompt)` | 漢字・カタカナ | 4択 | `words.json` の `kanji[poolName]` から |
| `makeOppGen(prompt)` | はんたいことば | 4択 | `words.json` の `opposites` から |
| `makeStoryGen(filter)` | 文章題 | 数値入力（単位付き） | `words.json` の `storyProblems` から |

### 出題オブジェクトの形式
```js
// 選択式
{ text: '問題文', choices: ['a','b','c','d'], ans: 2, html: true }
// 数値入力
{ text: '問題文', input: true, ans: '42', unit: 'こ', html: true }
```
`html: true` の場合、`text` はHTMLとして描画される（SVG埋め込み・`<span class="big">` 等に使用）。

### 重複排除
`buildQuestions()` が問題文＋選択肢のキーで重複を排除する（最大300回試行）。

## 5. 文章題エンジンの仕様

### テンプレート形式（`words.json` の `storyProblems[]`）
```json
{
  "id": "bird-sub",
  "pattern": "sub",
  "text": "とりが {a}わ います。{b}わ とんで いきました。のこりは なんわ？",
  "vars": { "a": [5, 15], "b": [1, "a-1"] },
  "unit": "わ"
}
```

### 変数解決のルール
- `vars` は宣言順に確定する（a を先に決めてから b の範囲を解決）
- 範囲の上下限には数値のほか **`"a"` / `"a-1"` 形式の参照のみ**が書ける
  （任意の式評価はセキュリティ・単純性の観点から**しない**）
- これにより「ひき算で b が a を超えない」＝答えがマイナスにならない制約が書ける

### 安全ガード（二重防御）
1. スキーマ側：`"b": [1, "a-1"]` のような従属関係で制約
2. 実行時：生成後に `0 <= answer <= 999` を検証し、違反したら**再生成（最大20回）**
3. 未知の `pattern` は `console.warn` してスキップ（エラーで止めない）
4. プレースホルダ未解決を検知したらスキップ

### 計算パターン（`PATTERNS`、層1）
現在 `add` / `sub` の2種のみ。**増やす場合はコード側に追加が必要**（`mul`/`div`/`diff` 等）。

## 6. デザインシステム

### 共通ルール
- 輪郭線：`#3D3323`（Ink Line）、`stroke-width: 1.5〜2.5`
- 塗り：明部・暗部の**2階調**（セルシェーディング）。グラデーション不使用
- 光源：右上45度で統一
- フォント：`'Hiragino Maru Gothic ProN','BIZ UDPGothic'`（丸ゴシック）

### エリア別カラーパレット

| エリア | ステージ | カード背景 | フレーム | 世界観 |
|---|---|---|---|---|
| みずべ | mori/hama/kawa | `#ECF2E0` | `#4E7E3F` | 明るい緑と水色の自然 |
| ちてい | douk/tani | `#E6E4EE` | `#5A5478` | 青紫の洞窟・鉱石・マグマ |
| ことば | iseki1/hashi/iseki2 | `#EFE7D2` | `#D4A63C` | 石造りの遺跡×黄金 |
| さんちょう | yama/shiro | `#E8EEF4` | `#5C6873` | 岩山・城・青灰色 |
| （ボス）とう | tou | `#EFEBF5` | `#8E7BC4` | 夜空・紫の神秘光 |
| そらの しま | sora1〜3 | `#EDF4FA` | `#5B9FC4` | 高空・雲・白い宮殿 |
| うみの そこ | umi1〜3 | `#E4F0EE` | `#2E7D99` | 深海・さんご・竜宮 |

**全テーマで文字コントラスト WCAG AA（4.5:1）以上を実測確認済み**（最低 8.57:1）。
新テーマ追加時も必ず実測すること。

### テーマの適用方法
`STAGES[i].theme` の値が `#scr-quiz` の `data-stage-theme` 属性に入り、
CSS が `[data-stage-theme="mori"]{ --card-bg:…; --card-frame:…; }` で変数を上書きする。
コーナー装飾（`.qcorner`）の背景SVGもエリアごとに差し替わる。

### マップ
- 島の上にステージノードをジグザグ配置（X座標は `XS[]` 配列、%指定）
- `AREAS[]` から**エリア帯（色ゾーン＋ラベル）を動的生成**。ステージ追加に自動追従する
- ノードの3状態：ロック（グレースケール）／挑戦中（`bd-arch` がglow）／クリア（`bd-arch` が金色＋★表示）
- 点線の道がノード間を結ぶ（ベジェ曲線）

## 7. 進捗データ（localStorage: `bk_state`）

```js
{
  stats: {                       // ステージIDごとの成績
    mori: { plays, best, bestN, tq, tc, cleared, last }
    //     プレイ回数, 最高星, -, 総問題数, 総正解数, クリア済み, 最終プレイ日
  },
  treasures: ['mori', 'hama'],   // 獲得済み宝物のステージID
  days: { '2026-07-13': { q: 20, c: 18 } },  // 日別の解答数・正解数
  buddyName: 'ピカリ',            // ナビキャラの名前
  introSeen: { mori: '2026-07-13' }  // 導入モーダルを見た日（同日は再表示しない）
}
```

**注意**：`STAGES[].id` と `bk_state` キーは進捗の永続化に直結する。**変更禁止**。

### バックアップ
親メニューから JSON エクスポート/インポート可能（機種変更・データ消去対策）。
形式：`{ app: 'bouken-drill', version: 1, savedAt: ISO文字列, state: {…} }`

## 9. words.json スキーマ

```json
{
  "version": 1,
  "kanji": {
    "g1a":  [["山","やま"], …],    // 1年配当・基本 30語
    "g1b":  [["王","おう"], …],    // 1年配当・応用 34語
    "kana": [["ぱん","パン"], …]   // カタカナ語 40語
  },
  "opposites": [["うえ","した"], …],  // 30ペア
  "storyProblems": [ … ]              // 12テンプレート（add 6 / sub 6）
}
```

**増やし方**：配列に要素を足すだけ。コード変更不要。
**新カテゴリ追加時**：`kanji` に新キー（例 `g2a`）を足し、STAGES で `makeKanjiGen('g2a', '…')` と参照。
その際 `WORDS_FALLBACK`（index.html内）にも最小限のデータを追加すること（file:// 対応のため）。
