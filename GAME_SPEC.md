実装の正は`index.html`です。島・ステージ一覧は`docs/stage_overview.md`、
デザイン判断は`REDESIGN_AUDIT.md`を正とします。

## 1. コンセプト

- 対象：小学1年生。進度に応じて2〜3年内容も追加する
- 形式：島ごとのカリキュラム × 海賊との冒険
- 学習ループ：行き先を選ぶ → 問題でバリアを削る → 宝を取り戻す → 次へ進む
- 本番端末：iPad Safari
- 配信：GitHub Pages上のPWA
- オフライン：必須

## 3. 画面構成

### ホーム／島選択

- 現在遊べる島をヒーローカードで表示
- クリア数、宝物数、次のステージを表示
- 「つづきから」で島マップへ進む
- 将来の島は小さな航路予告として表示

### 島マップ

- 島名、攻略率、次のミッション、次の宝物を上部に表示
- ステージノードはCLEAR / NEXT / OPEN / LOCKの4状態
- 各ノードに番号、ステージ名、学習スキル、星を表示
- クリア済みステージの次が通常の現在地
- 親メニューの確認モードでは全ステージを一時解放

### クイズ

- 上部にステージ名、敵、バリア、コンボ、問題番号を表示
- 60%正解に必要な回数をバリアとして表現
- 正解ごとにバリアゲージが進む
- 3連続以上でコンボ表示を強調
- 正解で「きらめき」を獲得
- 問題タイプに応じて回答領域を切り替える

### 結果

- 正答率、星、正答数、最大コンボ、きらめきを表示
- 初回クリア時は宝物発見演出
- クリア時の主導線は次のステージ
- 未クリア時の主導線は再挑戦
- あと何問でクリア／満点だったかを表示

### 図鑑

- 島ごとの宝物数と進捗バー
- 未取得でも宝物のシルエット、発見ステージ、必要な学習内容を表示
- 取得後は宝物名、メモ、身につけた力、敵コメントを表示

### 保護者画面

- PIN：`1234`
- 合計解答数、正答率、直近7日
- 得意な内容と見守りポイント
- ステージ別成績
- JSONバックアップ／復元
- コンパスの改名
- 確認モード
- 全データリセット

## 5. 問題タイプ

| type | 操作 |
|---|---|
| choice | 選択肢を1つ選ぶ |
| input | 画面テンキーで数値入力 |
| tapcount | 絵を順にタップして数え、数値入力 |
| fillblank | 式の□に入る数を入力 |
| match | 上段と下段から合計が指定数になるペアを選ぶ |
| dig | 正解が書かれた砂山を選ぶ |
| witch | 個数が異なる絵を選ぶ |

問題生成時は、同じ挑戦内と直前の挑戦との重複を避ける。
チャレンジ対象ステージでは、最後の1問を物語付き文章題に置き換える。

## 6. クリアと報酬

- クリア：60%以上正解
- ★★★：全問正解
- ★★☆：全問数−2問以上正解
- ★☆☆：クリアライン以上
- 初回クリア：宝物を獲得
- 正解時きらめき：基本10。連続正解に応じて加算
- コンボ：不正解で0に戻る

既存のステージID、島ID、`bk_state`は変更禁止。

## 8. 音

- Web Audio APIで効果音とチップチューンBGMを合成
- 外部音声ファイルなし
- iPad Safariの制約に合わせ、最初のユーザー操作後に再生開始
- 音ボタンは、効果音＋BGM／効果音のみ／すべてOFFの3段階

## 9. PWA

- HTMLと`words.json`はnetwork-first
- その他の静的資産はcache-first
- 新Service Worker検知時に更新トーストを表示
- 更新時は`sw.js`の`CACHE`番号を上げる

## 10. アクセシビリティ

- 主要ボタンのタップ領域は原則44px以上
- SVGだけの選択肢にも`aria-label`を付ける
- 状態を色だけで表現しない
- `prefers-reduced-motion`時はアニメーションを停止
- 子ども向け文言は、ひらがな中心で短くする
