# ぼうけんドリル v10 全体診断と刷新方針

作成日: 2026-07-17

## 1. 結論

現行v9は、学習範囲の分解、問題生成、進捗保存、PWA対応という「教材の土台」は良い。
一方で、画面ごとの機能を並べた状態に留まり、子どもが感じる一本のゲーム体験になっていない。

抜本改修では、問題数やイラスト量を単純に増やすのではなく、次のゲームループを全画面で一貫させる。

> 行き先を選ぶ → 敵の作戦を知る → 問題でバリアを削る → 宝を取り戻す → 島が豊かになる → 次へ進む

## 2. 現行版の良いところ

- 「島＝1カリキュラム」の分け方は、今後の拡張と学習の見通しに向いている。
- 11ステージの段階設計には、数える・分解・絵・式・確認テスト・繰り上がりという流れがある。
- tapcount / match / dig など、入力式以外の問題タイプを既に持っている。
- SVG、効果音、BGM、オフライン、進捗保存を外部依存なしで成立させている。
- 親向けの成績、バックアップ、確認モードまで運用機能が揃っている。

この土台は残し、見せ方・手応え・導線を作り直す。

## 3. 改善点の洗い出し

### A. 第一印象と世界観

1. ロゴが絵文字＋文字だけで、作品固有の顔になっていない。
2. 初回画面に島カードが並ぶだけで、何を取り戻す冒険なのかが視覚的に伝わらない。
3. coming soonカードが大きな灰色ブロックとして並び、最初から未完成感を強調している。
4. キャラクターは存在するが、画面内での役割と視線誘導が弱い。
5. 背景、カード、ボタンの深度差が小さく、すべて同じ平面上に見える。

### B. 島選択とマップ

1. 島選択は「選ぶ画面」ではなく、実質1件だけ押せる一覧になっている。
2. 今やるべきステージが、島カード上で分からない。
3. マップが非常に縦長で、ゴールや全体の距離を把握しにくい。
4. ステージ名は小さい白ラベルで、学習内容と報酬が見えない。
5. ロック済みノードの灰色面積が大きく、進みたい気持ちより閉塞感が勝つ。
6. 現在地のコンパスが小さく、プレイヤーの分身として機能していない。
7. エリア境界はあるが、景色が変化した体験になっていない。

### C. クイズと操作感

1. 問題カード、選択肢、テンキーが縦に積まれ、ゲームというよりフォーム入力に見える。
2. 上部の丸い進捗だけでは、何のために正解しているか分からない。
3. 正解・不正解の全画面オーバーレイが流れを止め、問題と冒険を分断する。
4. 正解しても敵・ステージ・宝物に変化が起きず、行為の結果が見えない。
5. 連続正解、自己ベスト、今回の上達など、プレイ中の短期目標がない。
6. tapcountでは数えるエリア、回答表示、テンキーの役割分担が曖昧で空の箱が見える。
7. SVG選択肢に読み上げ用ラベルがなく、アクセシビリティ上も弱い。
8. 「やめる」が左上に強く出ており、冒険継続より離脱が目立つ。

### D. フィードバックと報酬

1. 正解フィードバックが毎回ほぼ同じで、10問の中に展開がない。
2. 宝物はクリア後に初めて現れ、プレイ前の期待を作れていない。
3. 星は結果画面だけの評価で、途中の盛り上がりにつながらない。
4. クリア後の主ボタンが「もういちど」で、次へ進む導線が弱い。
5. 失敗時に、あと何問・どの力があれば成功するかが分からない。
6. 宝を取った後にマップや島の景色が豊かになる変化がない。

### E. 図鑑

1. 未取得時は11枚の同じ「？」で、集めたい対象を想像できない。
2. 宝物と学習スキルの関係が見えない。
3. レア度、発見場所、コレクションのまとまりがない。
4. 取得後もカードを眺める以外の遊びがない。

### F. 保護者画面

1. 子ども向け画面と同じ見た目で、管理画面として情報を読み取りにくい。
2. 正答率と回数はあるが、「どこでつまずいているか」の要約がない。
3. 固定PINの扱いや確認モードの状態が、運用機能として目立ちにくい。
4. ステージ一覧が長くなると、島追加時に見通しが悪くなる。

### G. 情報設計・文章

1. 初回ストーリーが一度に長文で表示され、読み始めるハードルが高い。
2. 画面見出し、説明、補助文のサイズ差が小さい。
3. 「島の目的」「今のミッション」「今回の報酬」が画面をまたいで引き継がれない。
4. ダジャレが世界設定として存在する一方、学習中の遊びには十分つながっていない。

### H. 技術・保守

1. README / GAME_SPEC / ROADMAP に旧17〜20ステージ仕様が残っている。
2. index.html内にも「全17ステージ」など旧コメントが残っている。
3. 旧v8の問題生成・BGM・SVGが多く残り、現行v9の中核が見つけにくい。
4. 常設の自動テストがなく、大改修時の回帰確認が手作業に依存する。
5. 画面要素に安定したテスト用属性がなく、操作検証が壊れやすい。

## 4. v10のデザイン原則

### 4-1. アートディレクション

- 方向性は「上質な絵本 × 海賊の冒険ボードゲーム」。
- 深い海色、明るい空色、珊瑚色、金色、生成りを主要色にする。
- 輪郭線付き2階調SVGは継承し、CSSの光・影・レイヤーで奥行きを足す。
- 絵文字は補助に限定し、主要なロゴ・状態・ボタンは固有の図形と文字で表現する。
- 子ども向けでも幼稚にせず、文字サイズとタップ領域は大きく保つ。

### 4-2. ゲームループ

- 各ステージを「ドリル10問」ではなく「敵のバリアを問題で壊すミッション」として見せる。
- 正解でバリアゲージを削り、連続正解でコンボと光の演出を強める。
- プレイ中に宝物のシルエットを見せ、目的を常に保つ。
- 結果画面では、正答数だけでなく最大コンボ・獲得したきらめき・次の行き先を示す。
- 初回クリア時は宝物発見、再挑戦時は自己ベスト更新を報酬にする。

### 4-3. 学習UX

- 不正解は罰ではなく、正解の見方を短く示す。
- 1問ごとの停止時間を短くし、テンポを上げる。
- 問題タイプごとに操作領域を明確にし、空の箱を出さない。
- 数える・分ける・式にする、という足場の違いを視覚的に見せる。
- 60%クリアは維持しつつ、次回はどこを伸ばせば星が増えるかを伝える。

## 5. 画面別の刷新内容

### ホーム／島選択

- アプリ固有の帆＋コンパスのロゴを作る。
- 現在の島を大きなヒーローカードとして表示する。
- 「つづきから」ボタンに次ステージ名・スキル・報酬を表示する。
- 全体進捗、称号、宝物数を同じカードにまとめる。
- coming soonは下部の小さな航路予告に変更する。

### 島マップ

- 上部に島名、攻略率、次のミッションをまとめた航海パネルを置く。
- ノードに番号、スキル、状態、星を持たせる。
- 現在地だけはキャラと「ここから！」を強く表示する。
- ロックは暗く塗りつぶさず、次の輪郭が見える霧表現にする。
- エリア背景の景色と色を明確に変える。

### クイズ

- ステージ名、敵、バリア、コンボ、問題数を1つのHUDにまとめる。
- 問題カードを主役にし、回答領域は問題タイプに合わせて切り替える。
- 正解でバリアが減り、画面内に小さな粒子とコンボ表示を出す。
- 不正解は正解を短く示し、バリアは維持して次へ進む。
- テンキーでは入力値を大きな回答ボックスに表示する。

### 結果

- 円形スコア、星、最大コンボ、きらめきを一目で見せる。
- 初回クリア時は宝物を大きく見せ、取得理由とスキルを添える。
- 主ボタンを「つぎへ すすむ」にし、再挑戦を副ボタンにする。
- 未クリア時は「あと○問」と具体的に示す。

### 図鑑

- 宝物のシルエットと発見場所を未取得時から見せる。
- 取得カードには宝物名、ステージ、身につけた力、船長コメントを整理して表示する。
- 島ごとのコレクション進捗を大きなバーで示す。

### 保護者画面

- PIN後は大人向けの白基調ダッシュボードへ切り替える。
- 最上部に「今週の学習」「得意」「次に見守るポイント」を要約する。
- 詳細表と運用機能を分離する。

## 6. 実装の優先順位

1. 全体トークン、ロゴ、ナビゲーション、ホームを刷新する。
2. マップの情報設計と現在地表現を刷新する。
3. クイズHUD、バリア、コンボ、回答領域、フィードバックを刷新する。
4. 結果画面と次ステージ導線を刷新する。
5. 図鑑と保護者画面を刷新する。
6. アクセシビリティ、レスポンシブ、オフライン、既存進捗を検証する。

## 7. 完了条件

- 初回30秒で「何をするゲームか」「どこを押すか」「何を取り戻すか」が分かる。
- 正解するたびに冒険が進んだことが画面上で分かる。
- クリア後に次の行動が迷わず選べる。
- 全画面が同じ作品の世界観と情報設計でつながっている。
- iPad縦向きで主要操作が片手でも届き、文字とタップ領域が十分大きい。
- 既存の進捗、問題生成、バックアップ、オフライン動作を壊さない。
- `prefers-reduced-motion`、キーボード操作、読み上げラベルに対応する。
