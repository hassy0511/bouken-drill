# コンセプト設計書 v6 — 「ダジャレかいぞくだんと 17の たから」

ぼうけんドリルに「なぜ冒険するのか」の物語と、笑いのキャラクターを導入する設計書。
**本書はコンセプト定義であり、実装は開発者のGO後に行う。**

---

## 1. コンセプト一言

> 島の17のたからものを盗んだ「ダジャレかいぞくだん」から、
> 問題を解いてたからを取り返す。船長は負けるたびにダジャレで負け惜しみを言う。

- 笑いの軸：**ことば遊び（ダジャレ）× 自信満々なのにポンコツな船長のギャップ**
- 下品要素：ゼロ（うんこ・おしり系の語彙は使わない）
- 既存の資産（17ステージ・宝物・ずかん・コンパス）は**一切変更せず**、意味付けを上から被せる

---

## 2. ダジャレ品質基準（セリフ執筆ルール）

過去に「ダジャレとして成立していない」例を出した反省から、基準を明文化する。
**セリフを書く際は必ず全項目を満たすこと。**

| # | 基準 | 説明 |
|---|---|---|
| 1 | **音の重なりが明確** | 同音または準同音が**最低2モーラ**重なること（例：こうかい／こうかい）。「なんとなく似ている」はNG |
| 2 | **両方の意味が子どもに分かる** | 掛けた2語とも小1が知っている語であること（「呆然」「肴」等はNG） |
| 3 | **ひらがなで読める** | 分かち書き・ひらがな中心。カタカナは既習語のみ |
| 4 | **文脈に乗っている** | そのステージ・宝物・場面に関係する語で掛けること |

### NG例（成立していないダジャレ）
- ✗「サメだけに シャレに ならんぜ」→「サメ」と「シャレ」に音の重なりがない
- ✗「わたさん! わたしだけにな!」→「わたさん」と「わたし」は掛詞として弱く、意味の二重性がない

### OK例（本書のセリフはすべてこの水準）
- ✓「こうかい するなよ? かいぞくだけに、**こうかい（航海）**なら まいにち してるけどな!」（後悔／航海：完全同音）
- ✓「もんだいが **とけた**? **とけい**だけに な!」（解けた／時計：3モーラ重複）

## 3. キャラクター

### キャプテン・ダジャーレ（新キャラ・敵役）

| 項目 | 設定 |
|---|---|
| 正体 | 「ダジャレかいぞくだん」の船長。自称「うみで いちばん おしゃれな かいぞく」 |
| 性格 | 自信満々・見栄っ張り・実はぜんぜん悪くなれない小物。負けるたびダジャレで強がる |
| 口癖 | 「**〜だけに な! がっはっは!**」（毎回この型で締める。くりかえしギャグ） |
| 怖さ | 控えめ。丸い体型・丸い目で「にくめない悪役」に振る（おしりたんていの敵と同水準） |

**ビジュアル指針**（実装時。既存SVG品質基準に完全準拠）：
- ずんぐり丸い体型＋赤い船長コート（明部・暗部の2階調）
- 三角帽子に「ダ」の一文字マーク（ドクロの代替。怖くない＋一目でダジャレ団と分かる）
- ヒゲがわかめのようにうねうねしている（海賊らしさ＋とぼけた印象）
- 輪郭線 `#3D3323`、光源右上45度、グラデーション不使用
- `BUDDY_SVG` と同方式で1つのSVG内に表情グループを持つ：
  `dj-face-normal`（ドヤ顔）／`dj-face-oops`（負け顔）／`dj-face-angry`（かんかん）

### コンパス（既存キャラ・続投）

- 子どもが命名済みのため**そのまま相棒として続投**（変更しない）
- 新しい役割：**ツッコミ役**。船長のダジャレに
  「【なまえ】「さむ〜い! うみが こおっちゃうよ!」」等と返す。
  “すべっても笑いになる”保険として機能する

### 子分（将来拡張枠・今回は実装しない）

- 候補：オウムの「**オウムがえし**」（名前自体がダジャレ。船長のセリフを微妙に間違えて繰り返す）
- エリアごとの子分（イカ＝**いかり**（怒り／錨）等）はコンテンツ拡充時の弾として温存

## 4. ストーリー全体（3幕構成）

### 第1幕：挑戦状（オープニング）
ある朝、島に挑戦状が届く。ダジャレかいぞくだんが17のたからを盗み、島中に隠した。
コンパスと一緒に取り返す旅に出る。

### 第2幕：追跡（#1〜#11）
みずべ→ちてい→ことば→さんちょうと、子分たちが隠したたからを順に奪還。
**さいごの とう（#11・既存ボス）で船長と初対決**。勝つと船長は海の底へ逃走。

### 第3幕：決戦（#12〜#17）
そらのしま経由で海底へ。**カタカナの ちんぼつせん（#16）＝かいぞくだんのアジト**。
**でんせつの りゅうぐう（#17・既存ボス）で最終決戦**。全たから奪還、島はお祭りに。
船長は改心して「また ダジャレしに くるぜ」と去る（悪者を完全に排除しない、優しい結末）。

### 既存進捗との整合（重要）
すでにクリア済みステージがある状態でこの物語を導入しても矛盾しないよう、
オープニングに分岐セリフを用意する：

> ダジャーレ「なにっ!? もう いくつか とりかえされてる だと!?
> や、やるじゃねえか…。だが のこりは ぜったいに わたさんぜ!」

進捗データ・ステージid・宝物システムは**一切変更しない**。

## 6. 実装スコープ（GO後の作業項目・参考）

1. `VILLAIN_SVG` 作成（表情3種、`BUDDY_SVG` と同方式）＋ `setVillainFace()` 相当
2. オープニング挑戦状モーダル（初回1回。既存進捗ありの場合の分岐セリフ）
3. 対象8ステージの `intro` に船長セリフを追加（既存の導入モーダルの仕組みに乗せる）
4. 結果画面に「負け惜しみ」1行を追加（ボスステージは専用セリフ）
5. 全クリア時のエンディング画面（ROADMAP P4「エンドコンテンツ」と統合して解決）
6. ずかんメモに船長コメント追記
7. セリフの格納場所の検討：頻繁に増やすなら `words.json` に `villainLines` を新設
   （2層分離の思想に合致。その場合 `WORDS_FALLBACK` にも最小セット必須）
8. `sw.js` の CACHE インクリメント

## 7. 検証観点（実装時）

- 全セリフがひらがな中心・分かち書きであること
- 掛詞の音の重なりを一覧表で機械的に確認（本書§2の基準）
- 子どもを否定する文言がないこと（船長の煽りも「よめないだろうけどな!」程度に留め、
  不正解時のフィードバックには船長を絡めない＝既存の励まし文言を維持）
- 怖さの目視確認（SVGをPNG化して確認。ドクロ・武器・とがった歯は描かない）
