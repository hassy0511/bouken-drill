## 0. 方向転換の要点（開発者の指示）

1. **教科を分ける**（算数と国語を混ぜない）
2. **教科の中もカリキュラム単位に分ける**（例：小1算数の「たしざん」「ひきざん」「文章題」…）
3. **カリキュラム単位で販売していく**構想
4. **問題のつくりを再検討**：計算するだけでなく、アクション要素・問題側のふざけを入れる
   （参考：うんこドリル小1たしざん。うんこ要素ではなく「問題のつくり」を参考にする）
5. ダジャーレ船長と戦うストーリーは継続

## 1. 参考書（うんこドリル たしざん）から抽出した設計原則

40ページを分析して得た、まねすべき「つくり」：

### 原則A：1冊=1スキル、でも出題の「型」は多彩
「たしざん」だけの本なのに飽きない理由は、同じスキルを**7〜8種類のインタラクション**で問うから。
- かぞえて かく／おなじ かずだけ ぬる
- 大きい ほうに ○
- **線むすび**（あわせて10になるペアを むすぶ）
- ならびかたの 穴うめ（1-2-□-4）
- 絵の スキャフォールドつき計算（●●＋●●●が がっちゃんこする図）
- 素の計算
- **逆算の 穴うめ**（□+8=10、「こたえが10になる しきを つくる」）
- 3つのかずの 計算
- **文章題（チャレンジ！）**：ふざけた シチュエーション＋「しき」と「こたえ」の2段回答

### 原則B：スキャフォールドの段階設計
新しい概念は必ず「**絵つき → 絵の補助が減る → 数字だけ → 逆算 → 文章題**」の順。
くり上がりは「10をつくる」手順そのものを図解（9はあと1で10→3を1と2にわける→…）。

### 原則C：定期的な「かくにんテスト」
5〜6単元ごとに、それまでの全タイプを混ぜた**採点つきテスト回**を挟む（1つ2てん等）。
→ 本ゲームでは**ボス戦（子分戦）に完全対応**させられる。

### 原則D：ページ単位の自己評価
毎ページ「まちがいが 0-2こ／3-5こ／6こ〜」の3段階評価 → 既存の★システムと一致。

### 原則E：アクション性のごほうびページ
単元の合間に**めいろ（正しい答えの道をたどる）**、**点つなぎ（1→20の順にむすぶ）**、
**「どれかな？」（見た目の違い探し）**が挟まる。ドリルの手を替え品を替えの正体はこれ。

### 原則F：問題側でふざける
文章題の状況が毎回ばかばかしい（「モンスターに うんこを 5こ なげました」等）。
**計算は真面目、状況はふざける**の分離が徹底している。
→ 本ゲームでは「ダジャーレかいぞくだん」の世界観の小道具（きんか・たから・カニ・タコ・
ダジャレ）でこれをやる。

## 2. 全体アーキテクチャ：「しま = カリキュラム」

### 2-1. 世界観の拡張（物語は継続・スケール）

> ダジャーレかいぞくだんが こんどは **「まなびの ぐんとう」**（群島）の たからを ぜんぶ ぬすんだ！
> しまごとに かいぞくだんの **かんぶ（子分キャラ）** が いすわっている。
> しまの もんだいを といて かんぶを おいはらい、たからを とりもどそう。
> ぜんぶの しまを すくうと、キャプテン・ダジャーレとの さいしゅうけっせんが まっている。

- **1つの しま = 1カリキュラム = 1販売単位**（例：「たしざんの しま」= 小1算数たしざん）
- 島ごとに**専属の子分キャラ**（SVG1体・性格・口ぐせのダジャレ）を置く
  → 「島を買う」＝「新しい敵と新しい宝と新しい景色が手に入る」という商品魅力になる
- キャプテン・ダジャーレ本人は**全体の黒幕**として挑戦状・全島クリアの大ボスに昇格
- コンパス（相棒）は全島共通で続投

### 2-2. 画面構成（3階層に再編）

```
[しまえらび]（新設・群島マップ）
   ├─ 島アイコンが海に浮かぶ。所持している島はカラー、未所持はシルエット＋「？」
   ├─ 島ごとに進捗リング（クリア済みステージ数）と子分の顔
   └─ 島をタップ
       └─ [しまマップ]（現行の縦スクロールマップを島単位で再利用）
           └─ [クイズ] → [結果]（現行フロー）
[ずかん]   … 島横断。たからを島別タブで表示
[おうちのひと] … 島別の成績＋（将来）購入管理
```

- 現行実装のマップエンジン・クイズエンジン・演出・BGM・進捗は**そのまま部品として使う**
- 変わるのは「STAGESが1本の配列」→「島ごとのSTAGES」になること

### 2-3. データ構造：カリキュラムパック

**シェル（共通エンジン）とパック（島データ）を分離する。** 現行の2層分離（ロジック/素材）を
3層に拡張：

```
層0 シェル      … index.html（エンジン・共通UI・船長・コンパス・BGM機構）
層1 出題ロジック … gen関数・問題タイプ（インタラクション）実装。全島共通のライブラリ
層2 カリキュラム … islands/<id>.json（下記スキーマ）＋語彙・テンプレデータ
```

```jsonc
// islands/math-g1-add.json（イメージ）
{
  "id": "math-g1-add",          // 不変ID（進捗キー）
  "title": "たしざんの しま",
  "subject": "math", "grade": 1,
  "henchman": {                  // 島の子分
    "name": "ハサミー",          // カニの幹部（例）
    "scene": "hasamy",           // SVGはシェル側レジストリに登録
    "catchphrase": "たしざんは たしかに おれさまの とくいわざ！"
  },
  "stages": [
    { "id": "add-s1", "name": "かぞえの いりえ",
      "skill": "count10",
      "mix": [ {"type":"tapcount","w":5}, {"type":"countwrite","w":5} ],
      "n": 10, "treasure": { ... }, "scene": "..." },
    ...
    { "id": "add-boss", "boss": true, "test": true, ... }
  ],
  "pools": { "storyProblems": [...], ... }   // 島固有の素材データ
}
```

- **進捗キーは `島id/ステージid`**。既存の `bk_state` は「はじまりのしま（レガシー島）」として
  そのまま読める形でマイグレーション（既存ユーザーの進捗・ずかんは消さない）
- 販売のロック管理は `state.owned = ['starter', 'math-g1-add', ...]`。
  **v9時点では全島アンロックでよい**（課金はネイティブ化後。ロックUIだけ先に作る）

### 2-4. 問題タイプ（インタラクション）の拡張

現行3種（4択／2択／テンキー入力）に、参考書由来の**アクション型を追加**：

| type | 内容 | 操作 | 実装難度 |
|---|---|---|---|
| `choice` | 選択（既存） | タップ | — |
| `input` | テンキー入力（既存） | タップ | — |
| `tapcount` | 「りんごを かぞえて すうじを おす」— 絵をタップすると印がつき、数えてから入力 | タップ×n＋入力 | 低 |
| `match` | 線むすび（あわせて10のペア等）。上下の札をタップでペア成立 | タップ×2×n | 中 |
| `fillblank` | 式の穴うめ（□+8=10、3+□=7） | テンキー | 低（inputの亜種） |
| `sequence` | 点つなぎ／ならびかた（つぎの かずを じゅんに タップ） | 連続タップ | 中 |
| `maze` | ぷちめいろ（分かれ道ごとに計算し、正しい方をタップして進む。3分岐=3問） | タップ×3 | 中 |
| `witch` | 「どれかな？」（条件に合う絵をえらぶ。かずが ちがう やつ等） | タップ | 低 |

- 1問=1画面の現行エンジンに乗る粒度に分解する（めいろも「分岐1つ＝1問」として内部処理し、
  見た目だけ連続させる）
- ドット進捗・正誤演出・重複防止（実体キー）は全タイプ共通で流用

### 2-5. ステージの標準構成（1島あたり）

参考書の構成をゲームに写像した標準テンプレート：

```
1島 = 8〜12ステージ
  [概念] → [絵つき練習] → [すうじ練習] → [逆算] → ★かくにんテスト(子分ミニ戦)
  → [発展1] → [発展2] → [アクション回(めいろ等)] → [文章題] → ★ボス(子分戦=まとめテスト)
```

- **かくにんテスト回**：それまでの全タイプ混合＋子分が登場して煽る。クリアで子分が一歩後退
- **ボス回**：島の総まとめ。勝つと子分が改心 or 逃走し、島の宝が全部戻る＋島クリア演出
- アクション回は「べんきょう感」の薄い回として意図的に配置（参考書のごほうびページに対応）

### 2-6. 教科分離

- 島は必ず単一教科。「さんすうの うみ」「こくごの うみ」で群島マップの海域を分ける
- 既存の混在20ステージは「**はじまりの しま**」（無料体験島・全教科ミックス）として残す
  → 体験版としての役割を持たせる。子どもの現行進捗もここに引き継がれて無駄にならない

## 4. 決めてもらいたい論点

1. 「はじまりのしま」方式（既存20ステージを体験島として残す）でよいか。
   代替案＝既存ステージを解体して各カリキュラム島に吸収（子の進捗の見え方が変わる）
2. 島の粒度：「たしざん」「ひきざん」を別売りにするか、「小1けいさん」で1島にするか
   （参考書は たしざん単体で1冊 → 本書は**単元ごと**を推奨）
3. 子分キャラのテイスト（各島1体・ダジャレ口ぐせ持ち）を出してよいか
