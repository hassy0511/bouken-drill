# ぼうけんドリル 〜まなびの ぐんとう〜（v10）

小学1年生を中心に、少し先の内容まで楽しく学べる子ども向け学習PWAです。
問題を解いて敵のバリアを壊し、島の宝物を取り戻しながらカリキュラムを進みます。

本番環境はiPad Safariです。ホーム画面に追加すると、縦向きの全画面アプリとして動きます。

## 現在遊べる内容

### たしざんの しま

小学校1年生の「たし算」を、11ステージで段階的に学びます。

1. 10までの数を数える
2. 5・7・10の数の分解
3. 絵を見ながらたし算
4. 10までのたし算
5. 確認テスト
6. 20までの数
7. 20までの繰り上がりなしのたし算
8. 3つの数のたし算
9. 答えの砂山を掘る遊び問題
10. 繰り上がりのたし算
11. まとめボス

「ひきざんの しま」「かんじの しま」「とけいの そらじま」は今後追加予定です。

## v10で刷新した体験

- ホームで、現在の進捗・次のステージ・次の宝物が分かる
- ステージを「ドリル」ではなく「敵のバリアを壊すミッション」として表示
- 正解するとバリアが減り、連続正解でコンボが増える
- プレイごとに「きらめき」を獲得
- 結果画面から次のステージへ直接進める
- 図鑑で未取得の宝物もシルエットと発見場所を確認できる
- 宝物と「身につけた力」が結びつく
- 保護者画面で、得意な内容と見守りポイントを要約する

診断内容と刷新方針は `REDESIGN_AUDIT.md` にまとめています。

## ファイル構成

| ファイル | 役割 |
|---|---|
| `index.html` | アプリ本体。HTML、CSS、JavaScript、SVG、BGMを内包 |
| `words.json` | 語彙・読解・文章題などの素材データ |
| `manifest.json` | PWA設定 |
| `sw.js` | オフラインキャッシュと更新処理 |
| `icon-192.png` / `icon-512.png` | ホーム画面アイコン |
| `GAME_SPEC.md` | 現行のゲーム仕様 |
| `REDESIGN_AUDIT.md` | v10の診断とデザイン原則 |
| `docs/stage_overview.md` | 島・ステージの正規一覧 |

## 技術方針

- バニラHTML / CSS / JavaScript
- ビルド工程なし
- 外部ライブラリなし
- 画像素材はインラインSVG
- 効果音・BGMはWeb Audio APIで合成
- 進捗は端末内の`localStorage`に保存
- オフライン動作対応
- `file://`直開き時は内蔵のフォールバック問題で動作

この構成は、GitHub Pagesへファイルを置くだけで配信できる運用を維持するための意図的な設計です。

## ローカル確認

ファイル直開きでも動作しますが、`words.json`とPWAを含めて確認する場合はローカルWebサーバー経由で開いてください。

例：

```text
python -m http.server 4173
```

その後、ブラウザで次を開きます。

```text
http://127.0.0.1:4173/
```

## GitHub Pagesでの公開

1. GitHubリポジトリの「Settings」を開く
2. 「Pages」を選ぶ
3. Sourceを「Deploy from a branch」にする
4. Branchを`main`、フォルダを`/(root)`にする
5. 表示されたURLをiPadのSafariで開く
6. 共有メニューから「ホーム画面に追加」する

公開URL：`https://hassy0511.github.io/bouken-drill/`

## 更新時の注意

コードまたは素材データを更新したときは、`sw.js`の`CACHE`番号を必ず上げます。

HTMLと`words.json`はnetwork-firstです。新しいService Workerを検知すると、アプリ内に
「あたらしい ぼうけんが とどいたよ！」と表示され、ボタン操作で最新版へ切り替わります。

## 進捗とバックアップ

- 保存キー：`bk_state`
- 現行スキーマ：v2
- 島ごとに成績・宝物・導入閲覧・エンディングを保存
- 親メニューからJSON形式で書き出し・読み込み可能
- ブラウザデータを消すと進捗も消えるため、機種変更前はバックアップ推奨

親メニューの初期PINは`1234`です。

## 開発上の重要事項

- 実装前に`CLAUDE.md`、`GAME_SPEC.md`、`REDESIGN_AUDIT.md`を読む
- 既存ステージIDと`bk_state`を変更しない
- 新しい問題カテゴリは`words.json`と内蔵フォールバックの両方を更新する
- 子どもを否定する文言を使わない
- 主要操作には読み上げ可能なラベルを付ける
- iPad縦向きと小画面の両方を確認する
- 変更後は全ステージ起動、問題形式、進捗、図鑑、親画面、Service Workerを検証する
