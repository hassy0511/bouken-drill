# ぼうけんドリル 〜まなびの しま〜 (v1)

子ども向け学習PWA。算数（1〜2年先取り）と国語（漢字よみ・はんたいことば）を
「島の冒険マップ」を進みながら学ぶ。iPadのホーム画面に追加して全画面アプリとして使う。

## ファイル構成

| ファイル | 役割 |
|---|---|
| index.html | アプリ本体（CSS/JS込みの単一ファイル） |
| manifest.json | PWA設定（アプリ名・アイコン・全画面表示） |
| sw.js | Service Worker（オフラインキャッシュ） |
| icon-192.png / icon-512.png | ホーム画面アイコン |

## 公開手順（GitHub Pages / 初回のみ・全部ブラウザで完結）

1. https://github.com で無料アカウントを作成（メール認証まで）
2. 右上「＋」→「New repository」
   - Repository name: `bouken-drill`（任意）
   - **Public** を選択（無料プランのPagesはPublicのみ）
   - 「Create repository」
3. リポジトリ画面の「uploading an existing file」リンク（または Add file → Upload files）
   - この5ファイルを**まとめてドラッグ&ドロップ** → 「Commit changes」
4. 「Settings」タブ → 左メニュー「Pages」
   - Source: **Deploy from a branch** / Branch: **main** / フォルダ: **/(root)** → Save
5. 1〜3分待つと Pages 画面上部に URL が表示される
   - `https://ユーザー名.github.io/bouken-drill/`

## iPad側の設定（初回のみ）

1. **Safari** で上記URLを開く
2. 共有ボタン（□↑）→「**ホーム画面に追加**」
3. 以後はホーム画面の 🏝️ アイコンから起動（全画面・オフラインOK）

## 更新方法

GitHubのリポジトリで index.html を差し替え（Upload filesで上書き）るだけ。
※Service Workerのキャッシュ更新のため、`sw.js` 内の `bouken-v1` を `v2, v3...` と
上げると確実に反映される。

## 運用メモ

- 親メニューのPIN: **1234**（v1では固定）
- 進捗は**iPad内（localStorage）にのみ保存**。他端末とは共有されない
- ブラウザデータを消す／Safariのプライベートブラウズで開くとリセットされる
- **バックアップ／復元**（親メニュー）：進捗をJSONファイルとして書き出し・読み込みできる。
  機種変更やデータ消去前に「⬇️ バックアップを保存」しておくと安全。書き出したファイルは
  iPadの「ファイル」アプリやiCloud Driveに保存される
- 進捗リセット: 親メニュー最下部
- 音は控えめの効果音のみ。右上 🔊 でON/OFF

### words.json（問題データの分離）
UIのコード（index.html）と問題の素材データ（words.json）を分離した。
**words.json を差し替えるだけで、コードを触らずに語彙・文章題を増やせる。**

| キー | 内容 | 増やし方 |
|---|---|---|
| `kanji.g1a` / `g1b` | 漢字の読み | 配列に `["漢字","よみ"]` を足す |
| `kanji.kana` | カタカナ語（40語） | 配列に `["ひらがな","カタカナ"]` を足す |
| `opposites` | はんたいことば（30ペア） | 配列に `["語","反対語"]` を足す |
| `storyProblems` | 文章題テンプレート（12種） | 下記の形式でオブジェクトを足す |

**文章題テンプレートの書き方**（数字が毎回変わり、答えも自動で連動する）：
```json
{
  "id": "apple-add",
  "pattern": "add",
  "text": "りんごが {a}こ あります。{b}こ もらいました。ぜんぶで なんこ？",
  "vars": { "a": [2, 9], "b": [1, 8] },
  "unit": "こ"
}
```
- `pattern`：`add`（たし算）／`sub`（ひき算）。※新パターンはコード側に追加が必要
- `vars`：変数の範囲 `[最小, 最大]`。上限に `"a-1"` と書くと「aより小さい」を表現でき、
  ひき算で答えがマイナスにならないようにできる
- 答えがマイナスや1000以上になる組み合わせは自動的に再生成される

**新しい漢字プールを足す場合**：`kanji` に新しいキー（例 `g2a`）を足し、
index.html の STAGES で `makeKanjiGen('g2a', 'プロンプト')` として参照する。

### 注意：PCでのファイル直開き（file://）について
ブラウザの制約により、`index.html` をダブルクリックで開いた場合は words.json を
読み込めない。この場合は **index.html 内の内蔵データ（少なめの語彙）で動作**する。
words.json の全量を使うには GitHub Pages 経由（https://）でアクセスすること。
